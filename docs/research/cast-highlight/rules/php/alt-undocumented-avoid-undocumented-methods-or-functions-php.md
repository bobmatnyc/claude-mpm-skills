# **Why you should care**

Documentation is useful as it helps maintaining and embedding the code well. Undocumented code causes problems in the code by decreasing agility and maintenance reduction.

# **Business Impacts**

Documenting code is advised as it allows for the code to be better maintained which frees up the code from potential bugs and reduces costs.
